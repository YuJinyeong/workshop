Spring Application: POJO(비즈니스 로직) +  설정 정보(애플리케이션 구성 정보) + 스프링 프레임워크

bean: *재사용*하기 위해서 스프링이 관리하는 객체

Model: Service + dao(spring: repo)

dto는 재사용을 안한다, stateful(<-> stateless)
	stateless: 상태를 유지하지 않는다.
	stateful: 상태를 유지한다.


annotation

빈 등록 방법
	1. xml 기반 - 태고부터 있다.
	2. @를 이용한 묵시적 DI --> 1 or 3과 함께 사용되어야 한다.
	    : 우리가 소스를 확보할 수 있을 때만 가능 --> 기존 자바 객체를 사용할 수 없다.
	3. java 기반 - spring 3.x~

	1, 3번 즉 명시적인 bean 선언은 우리가 만든 소스뿐 아니라 다른 라이브러리의 객체도 빈으로 관리 가능


*스프링의 DI 동작*
	1. 개발자는 빈으로 관리 될 POJO를 작성
	2. 개발자는 메타정보를 작성해서 스프링 컨테이너에 전달
	3. 스프링 컨테이너는 메타정보를 기반으로 POJO 객체 생성, setter/생성자를 호출해서 빈 설정 마무리
	4. 클라이언트 코드에서 컨테이너에게 빈 요청
	5. 컨테이너가 관리중인 빈 객체 반환
